# FASE 3 — 3.2 Endpoints da API REST

**Base URL:** `http://localhost:8080/api`
**Formato:** JSON (UTF-8)
**CORS:** habilitado para qualquer origem em `/api/**` (será restrito na Fase 4)

---

## Recurso: Produtos (`/api/produtos`)

| Método | Rota | Descrição | Body |
|---|---|---|---|
| GET | `/produtos` | Lista todos os produtos | — |
| GET | `/produtos?categoria=salgados` | Filtra por categoria | — |
| GET | `/produtos?nome=coxinha` | Busca por nome (parcial) | — |
| GET | `/produtos?apenasDisponiveis=true` | Lista somente disponíveis | — |
| GET | `/produtos/{id}` | Busca por ID | — |
| POST | `/produtos` | Cria um produto | `Produto` |
| PUT | `/produtos/{id}` | Atualiza um produto | `Produto` |
| DELETE | `/produtos/{id}` | Remove um produto | — |

### Exemplo — POST /api/produtos
```json
{
  "nome": "Pão de Queijo",
  "descricao": "Quentinho, mineiro tradicional",
  "preco": 4.50,
  "categoria": "salgados",
  "emoji": "🧀",
  "disponivel": true
}
```

## Recurso: Clientes (`/api/clientes`)

| Método | Rota | Descrição | Body |
|---|---|---|---|
| GET | `/clientes` | Lista clientes | — |
| GET | `/clientes/{id}` | Busca por ID | — |
| POST | `/clientes` | Cadastra um cliente | `Cliente` |
| PUT | `/clientes/{id}` | Atualiza um cliente | `Cliente` |
| DELETE | `/clientes/{id}` | Remove um cliente | — |

### Exemplo — POST /api/clientes
```json
{
  "nome": "Aluno Demo",
  "email": "demo@senac.br",
  "senha": "123456",
  "instituicao": "Senac São Paulo"
}
```

### Exemplo — POST /api/clientes/login
```json
{ "email": "demo@senac.br", "senha": "123456" }
```

## Recurso: Pedidos (`/api/pedidos`)

| Método | Rota | Descrição | Body |
|---|---|---|---|
| GET | `/pedidos` | Lista todos os pedidos | — |
| GET | `/pedidos?clienteId=1` | Lista pedidos de um cliente | — |
| GET | `/pedidos/{id}` | Busca pedido por ID | — |
| GET | `/pedidos/codigo/{codigo}` | Busca pedido pelo código de retirada | — |
| POST | `/pedidos` | Cria um novo pedido | `PedidoRequest` |
| PATCH | `/pedidos/{id}/status?status=PRONTO` | Atualiza o status | — |
| POST | `/pedidos/{id}/cancelar` | Cancela o pedido | — |
| DELETE | `/pedidos/{id}` | Remove o pedido | — |

### Exemplo — POST /api/pedidos
```json
{
  "clienteId": 1,
  "itens": [
    { "produtoId": 1, "quantidade": 2 },
    { "produtoId": 3, "quantidade": 1 }
  ]
}
```

### Resposta (201 Created)
```json
{
  "id": 7,
  "codigoRetirada": "ABC-1234",
  "cliente": { "id": 1, "nome": "Aluno Demo" },
  "itens": [
    { "id": 11, "produto": { "id": 1, "nome": "Coxinha Crocante" }, "quantidade": 2, "precoUnitario": 6.50 },
    { "id": 12, "produto": { "id": 3, "nome": "Refrigerante Lata" }, "quantidade": 1, "precoUnitario": 5.00 }
  ],
  "valorTotal": 18.00,
  "status": "AGUARDANDO_PAGAMENTO",
  "dataCriacao": "2026-04-26T15:42:11"
}
```

### Estados possíveis (`StatusPedido`)
`AGUARDANDO_PAGAMENTO` → `PAGO` → `EM_PREPARO` → `PRONTO` → `RETIRADO`
(qualquer estado pode ir para `CANCELADO`, exceto `RETIRADO`).

---

## Padrão de erros

Toda exceção tratada retorna um JSON no formato:
```json
{
  "timestamp": "2026-04-26T15:50:01",
  "status": 404,
  "erro": "Not Found",
  "mensagem": "Produto não encontrado com id: 99"
}
```

| Código | Quando ocorre |
|---|---|
| 400 | `RegraNegocioException` (ex.: e-mail já existe, item inválido) |
| 404 | `RecursoNaoEncontradoException` (ID inexistente) |
| 500 | Erro interno não tratado |

## Sincronização de dados (Frontend ⇄ Backend)

| Evento no frontend | Endpoint chamado |
|---|---|
| Abrir tela `produtos.html` | `GET /api/produtos` |
| Filtrar por categoria | `GET /api/produtos?categoria=...` |
| Submeter login (`index.html`) | `POST /api/clientes/login` |
| Submeter cadastro (`cadastro.html`) | `POST /api/clientes` |
| Confirmar pagamento (`retirada.html`) | `POST /api/pedidos` |

O carrinho permanece local (`localStorage`) durante a navegação e só é
sincronizado com o servidor no momento da finalização — uma única chamada
`POST /api/pedidos` cria o pedido e devolve o `codigoRetirada`.
