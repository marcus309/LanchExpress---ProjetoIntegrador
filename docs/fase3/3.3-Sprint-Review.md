## 1. Objetivo da Sprint

Entregar a funcionalidade central do aplicativo: o backend Spring Boot com o
**CRUD completo** dos dados de Produto, Cliente e Pedido, expostos via
**API REST**, e a integração do *frontend* (HTML/CSS/JS) com esses serviços
para sincronizar o cardápio e registrar pedidos.

## 3. Demonstração

Roteiro apresentado durante a Sprint Review:

1. Subida do servidor com `./mvnw spring-boot:run` — log do `DataLoader`.
2. Console H2 (`/h2-console`) mostrando as tabelas `tb_produto`, `tb_cliente`,
   `tb_pedido`, `tb_item_pedido`.
3. Acesso a `http://localhost:8080/html/index.html` — login com
   `demo@senac.br / 123456`.
4. Cardápio carregado dinamicamente da API (`GET /api/produtos`).
5. Filtro por categoria (chamada à API com `?categoria=`).
6. Adição de itens ao carrinho → `POST /api/pedidos` na finalização.
7. Tela de retirada mostrando o `codigoRetirada` retornado pelo backend.
8. Demonstração rápida via *Postman* dos verbos `PUT` e `DELETE` em `/api/produtos/{id}`.
