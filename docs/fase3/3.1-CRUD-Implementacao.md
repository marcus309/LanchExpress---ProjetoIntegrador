## 2. Modelo de dados (resumo)

| Tabela | Campos principais | Relacionamento |
|---|---|---|
| `tb_produto` | id, nome, descricao, preco, categoria, emoji, disponivel | 1:N com `tb_item_pedido` |
| `tb_cliente` | id, nome, email (único), senha, instituicao | 1:N com `tb_pedido` |
| `tb_pedido` | id, codigoRetirada (único), valorTotal, status, dataCriacao | N:1 com `tb_cliente`; 1:N com `tb_item_pedido` |
| `tb_item_pedido` | id, quantidade, precoUnitario | N:1 com `tb_pedido` e `tb_produto` |

## 3. Operações CRUD por entidade

### Produto
- **Create:** `POST /api/produtos`
- **Read:**  `GET /api/produtos` (com filtros opcionais), `GET /api/produtos/{id}`
- **Update:** `PUT /api/produtos/{id}`
- **Delete:** `DELETE /api/produtos/{id}`

### Cliente
- **Create:** `POST /api/clientes` (valida e-mail duplicado)
- **Read:**  `GET /api/clientes`, `GET /api/clientes/{id}`
- **Update:** `PUT /api/clientes/{id}`
- **Delete:** `DELETE /api/clientes/{id}`
- **Extra:** `POST /api/clientes/login` para autenticação simples

### Pedido
- **Create:** `POST /api/pedidos` (com lista de itens — gera código de retirada)
- **Read:**  `GET /api/pedidos`, `/{id}`, `/codigo/{codigo}`
- **Update:** `PATCH /api/pedidos/{id}/status?status=PRONTO`
- **Delete:** `DELETE /api/pedidos/{id}` ou `POST /api/pedidos/{id}/cancelar`

## 4. Regras de negócio aplicadas

1. E-mail de Cliente é **único** (`existsByEmail`).
2. Pedido sempre gera um **código de retirada único** (3 letras + 4 dígitos).
3. Não é possível criar um pedido sem itens.
4. Não é possível pedir produto **indisponível**.
5. Pedido com status `RETIRADO` não pode ser cancelado.
6. `valorTotal` do pedido é calculado a partir do `precoUnitario` × `quantidade`
   de cada `ItemPedido` (snapshot do preço no momento da compra).

## 5. Como testar localmente

```bash
# Subir o servidor
./mvnw spring-boot:run

# (em outro terminal) Listar produtos
curl http://localhost:8080/api/produtos

# Criar pedido
curl -X POST http://localhost:8080/api/pedidos \
  -H "Content-Type: application/json" \
  -d '{ "clienteId": 1, "itens": [ {"produtoId":1,"quantidade":2} ] }'

# Console do banco H2
# acesse http://localhost:8080/h2-console
# JDBC URL: jdbc:h2:mem:lanchexpressdb  | usuário: sa | senha: (vazia)
```
